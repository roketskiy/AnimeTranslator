# AnimeTranslator Textual TUI 设计文档

**日期**: 2026-04-22
**状态**: 已确认
**作者**: OpenCode

---

## 1. 目标

为 AnimeTranslator 设计并实现一个基于 **Textual** 框架的终端用户界面（TUI），参考 HydroToDo 的界面风格，提供向导式的翻译流程体验。

### 非目标

- 不改动现有的 `click` CLI（`core_cli.py`），保留供脚本和自动化使用。
- 不改动核心翻译逻辑（`core/translator.py`、`core/subtitle_parser.py` 等）。
- 不增加自然语言对话功能。

## 2. 视觉风格

### 2.1 配色方案

| 元素 | 色值 | 用途 |
|------|------|------|
| 主强调（品牌色） | `#d44020` | ASCII Banner、选中项、进度条 |
| 背景 | `#1e1e1e` | 全局背景 |
| 步骤指示 | `#ffd700` | 步骤编号 |
| 次要文字 | `#a0a0a0` | 辅助信息、时间戳 |
| 成功 | `#4caf50` | 翻译完成、成功提示 |
| 错误 | `#f44336` | 错误提示 |
| 边框/分隔线 | `#333333` | 面板边框、分隔线 |
| 白色 | `#ffffff` | 主要文字、步骤名称 |

### 2.2 布局结构

垂直 4 层结构（无底部常驻操作栏）：

```
+------------------------------------------+
|  ASCII Banner (橙红 #d44020)             |  ← 7-8 行，居中
+------------------------------------------+
|  Step 2/4  ──  选择源语言                 |  ← 步骤指示器
+------------------------------------------+
|                                          |
|  主内容区 (Content)                       |  ← 当前步骤交互控件
|                                          |
+------------------------------------------+
|  ─────────────────────────────────────── |
|  预览面板 (Preview Panel)                 |  ← 已选文件元数据/参数
+------------------------------------------+
```

**响应式处理**:

- 小屏幕（< 40 行高）：预览面板可折叠（按 `p` 切换显示/隐藏）。
- 极小屏幕（< 25 行高）：隐藏 Banner，只保留步骤条 + 内容区。
- 终端过小（< 80 列或 < 30 行）：显示警告信息，暂停交互，等待窗口调整。

### 2.3 ASCII Banner

复用现有 `main.py` 中的 `_render_banner()` 和 `_render_shadow()` 逻辑：

- **实心层**：`ANIME` + `TRANS`，使用 `#d44020`（ANSI `38;2;212;64;32m`）。
- **空心阴影层**：同字母，用 `░`（light shade）字符拼出，向右下方错位 1 个字符。
- **叠加方式**：`_merge_banner_line()` 逐字符合并两层，实心层字符优先覆盖阴影层，形成 3D 错位感。

Banner 在向导各步骤中**常驻显示**在顶部。

## 3. 导航方式：向导式（Wizard）

三种翻译模式各自拥有独立的向导流程。

### 3.1 模式 1：翻译字幕文件 (`.srt` / `.ass`)

| 步骤 | 界面内容 | 预览面板显示 |
|------|---------|-------------|
| **1/4** 选择文件 | 文件浏览器（过滤 `.srt` `.ass`）或路径输入框 | 空 |
| **2/4** 源语言 | 选项列表：日本語 / English | 文件名、大小 |
| **3/4** 输出选项 | 是否保留原轨（是/否）、输出路径输入（带默认值） | 文件名、大小、源语言 |
| **4/4** 执行 | 实时进度条、批次翻译统计、预计剩余时间 | 所有参数汇总 |

### 3.2 模式 2：翻译视频软字幕

| 步骤 | 界面内容 | 预览面板显示 |
|------|---------|-------------|
| **1/5** 选择视频 | 文件浏览器（过滤 `.mp4` `.mkv` `.avi` `.mov` `.ts`） | 空 |
| **2/5** 选择轨道 | 列出检测到的字幕轨道（索引、编码、语言、标题） | 视频文件名、时长 |
| **3/5** 源语言 | 日本語 / English | 视频信息、轨道信息 |
| **4/5** 输出选项 | 是否保留原轨、输出路径 | 以上 + 源语言 |
| **5/5** 执行 | 先提取字幕 → 再翻译，显示两阶段进度 | 所有参数汇总 |

### 3.3 模式 3：翻译视频硬字幕 (OCR)

| 步骤 | 界面内容 | 预览面板显示 |
|------|---------|-------------|
| **1/4** 选择视频 | 文件浏览器 | 空 |
| **2/4** 源语言 | 日本語 / English | 视频文件名、时长 |
| **3/4** 输出选项 | 输出路径、是否双语输出 | 以上 + 源语言 |
| **4/4** 执行 | OCR 识别进度 + 翻译进度，两阶段进度条 | 所有参数汇总 |

## 4. 交互规则

### 4.1 全局快捷键

| 按键 | 行为 |
|------|------|
| `↑/↓` | 在选项/文件列表中导航 |
| `Enter` | 确认当前选择 |
| `Esc` | 返回上一步（第一步时返回到模式选择页） |
| `Tab` | 在文件浏览器和路径输入框之间切换 |
| `h` / `?` | 弹出帮助面板，显示当前步骤可用的快捷键 |
| `q` | 退出应用（仅在模式选择页有效，其他页按 Esc 逐步返回） |
| `p` | 切换预览面板的显示/隐藏 |
| `Ctrl+C` | 强制退出 |

### 4.2 帮助面板

界面**无任何常驻快捷键提示**。按 `h` 或 `?` 时弹出模态帮助面板，覆盖在主界面上方：

```
+------------------------------------------+
|  帮助                        [Esc] 关闭   |
+------------------------------------------+
|  全局                                      |
|    ↑/↓        导航选项                    |
|    Enter      确认                        |
|    Esc        返回上一步                  |
|    Tab        切换文件浏览器/输入框       |
|    p          切换预览面板                |
|    q          退出（仅在首页）            |
|                                           |
|  当前步骤: 选择源语言                      |
|    ↑/↓        选择语言                    |
|    Enter      确认选择                    |
+------------------------------------------+
```

帮助面板内容**根据当前步骤动态变化**，只显示当前上下文相关的快捷键。

## 5. 组件架构

### 5.1 目录结构

```
ui/
├── app.py                      # Textual App 主入口，路由管理
├── screens/
│   ├── welcome.py              # 欢迎页：模式选择仪表盘（3 个卡片）
│   ├── wizard.py               # 向导主框架：Banner + 步骤条 + 内容区 + 预览
│   └── help.py                 # 帮助浮层面板
├── widgets/
│   ├── banner.py               # ASCII 像素大字 Banner（复用现有逻辑）
│   ├── step_indicator.py       # 步骤指示器：Step X/Y ── 步骤名称
│   ├── file_browser.py         # 文件浏览器：DirectoryTree 封装，过滤扩展名
│   ├── option_list.py          # 选项列表：单选，↑/↓ 导航，Enter 确认
│   ├── preview_panel.py        # 预览面板：文件元数据、已收集参数
│   ├── progress_view.py        # 翻译进度：进度条 + 批次统计 + 预计时间
│   └── input_panel.py          # 输入组件：路径输入，带默认值，支持浏览
└── styles.tcss                 # Textual CSS 样式定义
```

### 5.2 各组件职责

| 组件 | 职责 | 依赖 |
|------|------|------|
| `AnimeTranslatorApp` | 应用入口、路由（Welcome ↔ Wizard ↔ Help）、全局事件路由 | Textual `App` |
| `WelcomeScreen` | 模式选择仪表盘：3 个大卡片（字幕文件 / 软字幕 / 硬字幕 OCR） | `Banner` |
| `WizardScreen` | 向导框架：固定 Banner + 步骤条，动态挂载内容区组件 | `Banner`, `StepIndicator`, `PreviewPanel` |
| `HelpScreen` | 模态帮助面板：覆盖显示，动态生成当前步骤的快捷键说明 | 无 |
| `Banner` | 渲染 ASCII 像素大字 + 3D 阴影效果 | 无 |
| `StepIndicator` | 显示 `Step X/Y ── 步骤名称`，当前步骤高亮 | 无 |
| `FileBrowser` | 基于 `DirectoryTree`，过滤指定扩展名，`Enter` 选中，`Tab` 切换输入模式 | `DirectoryTree` |
| `OptionList` | 单选列表，`↑/↓` 导航，`Enter` 确认，选中项橙红高亮 | `ListView` |
| `PreviewPanel` | 显示已收集的参数和文件元数据，随步骤动态更新 | 无 |
| `ProgressView` | 实时进度条、批次统计、预计剩余时间 | `ProgressBar` |
| `InputPanel` | 带默认值的文本输入，支持 `tkinter.filedialog` 浏览 | 无 |

## 6. 数据流与状态管理

### 6.1 状态定义

`WizardScreen` 使用 Textual 的 **reactive attributes** 管理向导状态：

```python
class WizardScreen(Screen):
    mode: reactive[str] = ""           # "file" | "soft" | "hard"
    step: reactive[int] = 1            # 当前步骤（1-based）
    total_steps: reactive[int] = 4     # 总步骤数（根据模式动态设置）
    
    # 收集的参数
    file_path: reactive[str] = ""
    source_lang: reactive[str] = "ja"
    keep_original: reactive[bool] = True
    output_path: reactive[str] = ""
    selected_track: reactive[int | None] = None
    is_bilingual: reactive[bool] = False
```

### 6.2 数据流

```
用户操作
  ↓
内容区组件（FileBrowser / OptionList / InputPanel）
  ↓  post_message(ParamSelected)
WizardScreen（更新 reactive state）
  ↓  state 变化自动触发
PreviewPanel（刷新显示）
  ↓  用户按 Enter / →
WizardScreen（step += 1，挂载下一步的内容组件）
```

### 6.3 预览面板更新规则

| 步骤 | 预览面板显示内容 |
|------|----------------|
| Step 1 后 | 文件名、文件大小 |
| Step 2 后 | + 源语言 |
| Step 3 后 | + 输出路径、是否保留原轨（或双语输出） |
| Step 4/5 后 | 所有参数汇总 + 实时进度条 |

## 7. 错误处理与边界情况

| 场景 | 处理方式 |
|------|---------|
| 终端过小（< 80 列或 < 30 行） | 全屏显示警告：`"当前分辨率: WxH | 最小: 80x30"`，暂停交互，等待窗口调整后自动刷新 |
| API Key 未设置 | Step 执行前检查，弹出错误面板：`[1] 我已设置，重试` / `[Esc] 返回` |
| 文件不存在 / 格式不支持 | 当前步骤红色错误提示，不进入下一步，允许用户重新选择 |
| 目录权限不足（文件浏览） | 灰色显示不可访问目录，自动跳过，不报错 |
| API 超时 / 速率限制 | 自动重试（复用 `config.TRANSLATE_RETRY_TIMES`），失败后显示 `[r] 重试 [s] 跳过本条 [Esc] 取消` |
| 翻译中按 Esc | 发送取消信号，Worker 在当前批次完成后优雅退出，提示"翻译已取消"，不保存部分结果 |
| 未知异常 | 显示简略错误信息，`[Esc] 返回首页` / `[q] 退出`，同时写日志文件 |

## 8. 与现有 CLI 的集成

`main.py` 入口逻辑保持不变：

```python
def main():
    if len(sys.argv) > 1:
        from core_cli import cli
        cli()
    else:
        from ui.app import AnimeTranslatorApp
        app = AnimeTranslatorApp()
        app.run()
```

### main.py 的改动

- 移除 `print_banner()`、`interactive_loop()`、所有 `prompt_*` 函数、`_LETTERS` 等 Banner 相关代码。
- 移除 `mode_file()`、`mode_soft()`、`mode_hard()` 中的业务逻辑（迁移到 `core/workflow.py`，供 TUI 和 CLI 共用）。
- 保留 `check_api_key()` 等通用工具函数。

### core_cli.py 的处理

- 完全保留，不改动任何接口。
- 底层翻译逻辑统一调用 `core/workflow.py` 中的函数，避免 TUI 和 CLI 各自维护两套翻译流程。

## 11. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| Textual 在 Windows 终端兼容性问题 | 高 | 早期在目标环境测试；保留 `tkinter.filedialog` 作为文件选择备用方案 |
| Worker 线程与 UI 状态同步复杂 | 中 | 统一使用 `post_message` 事件机制，避免直接操作 UI；翻译在后台线程执行 |
| 状态机逻辑膨胀 | 中 | 每个状态（步骤）独立处理函数，避免嵌套条件；状态转换集中维护在 `WizardScreen` |
| 文件浏览器性能（大目录） | 低 | 使用 `DirectoryTree` 懒加载，或限制初始显示深度 |
